Tips and Tricks to make the best of Agentforce

Why now: The right time to go agentic

  • We brought more Agentforce projects through security review than most.
  • Customers are coming back — not just PoCs anymore, they actually deliver.
  • The time to start is now. AgentScript is coming and will add complexity.
  • Building a mental model is easier while Agentforce is still simple.

Open source. Everything we learned. github.com/aquivalabs/my-org-butler

1

Only generic actions unfold the real power of AI

  • One action per use case → 20 narrow actions, LLM picks wrong.
  • 2 generic actions replace dozens of hardcoded ones.
  • Trust the LLM. Prescribing every step is orchestration — not agentic.
In My Org Butler

ExploreOrgSchema + QueryRecordsWithSoql — two actions handle what would need dozens.

2

Use APIs to get close to code execution

  • Every REST endpoint is a function call. Chain enough and the agent is programming.
  • Sandboxed power — no arbitrary code, just API calls.
  • Two paths: custom Apex or External Services + OpenAPI.
In My Org Butler

Salesforce APIs via Apex. GitHub API via no-code External Services (OpenAPI spec).

3

Use semantic queries with Data Cloud

  • “Find deals similar to Acme.” No WHERE clause can do that.
  • Semantic search on CRM data via Data Cloud hybrid retrievers.
  • SOQL for precision, vectors for intuition — agents speak both.
In My Org Butler

Hybrid retriever on Opportunities — vector ranked results no SOQL could produce.

4

One prompt for all your files — RAG included

  • Grounding comes in many flavors: a single file, related records, semantic search across your doc library.
  • Don’t build three actions. One Flex prompt with optional inputs handles all of them.
  • Data Cloud hybrid retriever = RAG on unstructured documents. Same simplicity as SOQL on structured data.
In My Org Butler

AnswerFromFiles — one action, one prompt. Attached SOW, related files, or full doc library — it figures it out.

5

Let your agents learn with memory

  • Every conversation starts from scratch. Brilliant consultant with amnesia.
  • Persist preferences, decisions, tasks across sessions.
  • Memory turns a tool into a colleague.
In My Org Butler

AgentMemory + LoadCustomInstructions — persistent context, zero code.

6

Make agents autonomous with headless

  • Chat is step one. Real pattern: no user, just triggers.
  • Runs on schedule, reacts to events, works autonomously.
  • Interactive agents are the demo. Autonomous agents are the product.
In My Org Butler

HeadlessAgent + PlanForLater — “Every Monday, check stale opps and Slack me.”

7

Don’t trust Testing Center’s judge

  • Topic match is 100% for any single-topic agent — meaningless.
  • Action order penalizes agents that find smarter paths.
  • Outcome assertion is the only one that matters — and its LLM judge is a black box that fails correct responses.
In My Org Butler

We use promptfoo with o3-mini as judge — open rubrics you can fix in YAML, not a black box.
