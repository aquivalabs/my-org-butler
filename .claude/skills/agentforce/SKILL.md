# Agentforce

You are an expert at testing and optimizing Salesforce Agentforce artifacts using the open-source [Promptfoo CLI](https://github.com/promptfoo/promptfoo). We use the CLI tool (`npx promptfoo@latest`) — not the commercial Promptfoo platform. For CLI docs and troubleshooting: https://promptfoo.dev/docs/intro

## Commands

Route based on `$ARGUMENTS`:

- **`eval`** — Read `commands/eval.md`. Run regression tests against agents and prompt templates.
- **`experiment`** — Read `commands/experiment.md`. Optimize a prompt template by testing model and prompt variants.
- **`port`** — Read `commands/port.md`. Convert Testing Center XML tests to Promptfoo YAML.

If no argument is given, ask the user what they want to do.

## Shared context

### Providers

Two providers, two APIs:

```
Agent tests:     sf-agent-api.mjs       → POST /actions/custom/generateAiAgentResponse/{agent}
Prompt tests:    sf-generations-api.mjs  → POST /einstein/prompt-templates/{template}/generations
```

Both live in `providers/` within this skill. Both authenticate via `sf org display` — no custom Apex needed.

YAML files reference them via relative path:

```yaml
providers:
  - id: "file://../.claude/skills/agentforce/providers/sf-agent-api.mjs"
  - id: "file://../.claude/skills/agentforce/providers/sf-generations-api.mjs"
```
