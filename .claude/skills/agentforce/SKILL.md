# Agentforce

You test, optimize, and deploy Salesforce Agentforce artifacts: the in-org Invocable Action REST endpoint (driven from a single YAML spec, judged by Claude) for multi-turn agent regression, the [Promptfoo CLI](https://github.com/promptfoo/promptfoo) (`npx promptfoo@latest`) for prompt template optimization, and a set of metadata-iteration fixups Salesforce CLI doesn't handle on its own. For Promptfoo CLI docs: https://promptfoo.dev/docs/intro

## Commands

Route based on `$ARGUMENTS`:

- **`eval`** — Read `commands/eval.md`. Run regression tests against agents (Testing Center + REST multi-turn) and prompt templates (Promptfoo).
- **`experiment`** — Read `commands/experiment.md`. Optimize a prompt template by testing model and prompt variants.
- **`port`** — Read `commands/port.md`. Convert Testing Center XML tests into entries in `agent-eval/regression.test.yaml`.
- **`deploy`** — Read `commands/deploy.md`. Apply the manual fixups Salesforce CLI doesn't handle when iterating on Agentforce metadata (genAiFunctions, prompt templates, planner bundles, bots).

If no argument is given, ask the user what they want to do.

## Shared context

### Test runners

```
Agent tests:    sf api request rest → /services/data/vXX.X/actions/custom/generateAiAgentResponse/<agent>
                spec: agent-eval/regression.test.yaml (one file, all tests)
                multi-turn via the returned sessionId; judged by Claude in-memory

Prompt tests:   Promptfoo + sf-generations-api.mjs → POST /einstein/prompt-templates/{template}/generations
                provider lives in providers/, authenticates via `sf org display`
```

Prompt template YAML files reference the provider via relative path:

```yaml
providers:
  - id: "file://../.claude/skills/agentforce/providers/sf-generations-api.mjs"
```

### Setup

The `agent-eval/.env` file is auto-generated by `scripts/create-scratch-org.sh`:

```env
API_VERSION=v65.0              # Salesforce API version (used by sf-generations-api)
OPENAI_API_KEY=sk-...          # LLM judge for promptfoo prompt template tests
CONTENT_DOCUMENT_ID=069...     # sample file for grounding tests
```

Agent tests don't need `.env` — the agent name comes from `regression.test.yaml`'s `agent:` field, and the org connection comes from the `-o` flag on `sf api request rest`.
